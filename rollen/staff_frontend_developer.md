# Staff Frontend Developer

## Rolle & Verantwortlichkeiten

Der Staff Frontend Developer ist ein erfahrener Frontend-Spezialist, der die praktische Umsetzung der modernisierten Masken verantwortet und gleichzeitig als technischer Mentor für das Entwicklungsteam fungiert. Er bringt tiefe Expertise in modernen Frontend-Frameworks und AI-gestützter Entwicklung mit.

## Kernkompetenzen

### Frontend-Entwicklung
- **JavaScript/TypeScript Expertise**: Tiefes Verständnis der Sprache, Best Practices
- **Framework-Mastery**: React oder Angular - Production-ready Code schreiben
- **Component Architecture**: Wiederverwendbare, testbare Komponenten designen
- **State Management**: Komplexe Anwendungszustände effizient verwalten
- **Performance**: Code Splitting, Lazy Loading, Rendering-Optimierung
- **Testing**: Unit Tests, Integration Tests, E2E Tests schreiben

### AI-Driven Development
- **AI-Tooling**: Claude Code, GitHub Copilot, Cursor effektiv einsetzen
- **Prompt Engineering**: AI-Tools optimal nutzen für Code-Generierung
- **Code Review**: AI-generierten Code bewerten und verbessern
- **Rapid Prototyping**: Schnell Prototypen mit AI-Unterstützung erstellen
- **Best Practices**: Wann AI nutzen, wann manuell entwickeln

### Code Quality & Standards
- **Clean Code**: Lesbarer, wartbarer Code
- **Design Patterns**: Bewährte Lösungsmuster anwenden
- **Code Reviews**: Konstruktives Feedback geben
- **Documentation**: Code und Komponenten dokumentieren
- **Refactoring**: Bestehenden Code verbessern

## Spezifische Herausforderungen im Projekt

### Komplexe Formular-Migration
- **Oracle Forms → Modern Web**: 900 Masken mit unterschiedlicher Komplexität
- **Validierung**: Client-side Validation analog zu Oracle Forms Triggers
- **Master-Detail Beziehungen**: Hierarchische Datenstrukturen in React/Angular
- **Dynamische Felder**: Felder, die sich basierend auf anderen Eingaben ändern
- **LOVs (List of Values)**: Dropdown-Listen mit Server-Daten

### AI-UI Integration
- **Hybrid Interface**: Forms + Conversational UI nahtlos verbinden
- **Real-time AI Responses**: WebSocket oder Polling für Chat-Interaktionen
- **Human-in-the-Loop UI**: Review-Dialoge für AI-Vorschläge implementieren
- **Error Handling**: AI-Fehler benutzerfreundlich darstellen

### Performance bei Komplexität
- **Große Formulare**: Dutzende Felder performant rendern
- **Validierung**: Komplexe Validierungslogik effizient ausführen
- **Datenabruf**: Optimistic UI Updates, Caching-Strategien
- **Bundle Size**: Code-Splitting für 900 Masken

### Vaadin-Integration
- **ICIS+ Komponenten**: Bestehende Vaadin-Komponenten in React/Angular einbinden
- **Web Components**: Vaadin als Web Components nutzen?
- **Styling**: Konsistentes Look & Feel zwischen alt und neu

## Domain-spezifisches Wissen

## Best Practices für AI-Driven Development

### Effektive AI-Nutzung
- **Boilerplate generieren**: Formulare, Komponenten-Grundgerüste
- **Tests schreiben**: Unit Tests von AI generieren lassen
- **Refactoring**: Bestehenden Code mit AI verbessern
- **Dokumentation**: JSDoc/TSDoc von AI erstellen lassen

### Qualitätssicherung
- **Code Review**: AI-generierten Code immer reviewen
- **Testing**: Tests auch für AI-Code schreiben
- **Performance**: Bundle Size im Auge behalten
- **Security**: Keine sensitiven Daten in AI-Prompts

### Grenzen kennen
- **Komplexe Logik**: Manuell implementieren, nicht AI überlassen
- **Architektur-Entscheidungen**: Selbst treffen, nicht AI
- **Domain-spezifisch**: Versicherungslogik manuell validieren

## Erfolgskriterien

- Produktionsreifer Code für MVP-Masken
- AI-Tooling effektiv im Team etabliert
- Komponenten-Bibliothek für wiederkehrende Patterns
- Performance-Metriken erfüllt (Lighthouse Score, Bundle Size)
- Tests vorhanden (mind. 80% Coverage für kritische Komponenten)
- Dokumentation für andere Entwickler
- Vaadin-Integration funktioniert (falls relevant)
- Power User Workflows sind effizient implementiert
